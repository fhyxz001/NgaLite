## 1. 项目概述

NGA 论坛轻量客户端，基于 Kotlin + Jetpack Compose 的原生 Android 应用。
初版仅实现「帖子列表 → 帖子详情」两级浏览；登录支持账号密码登录（RSA 加密），粘贴 Cookie 作为兜底。

## 2. 技术选型

## 3. 功能范围

仅以下四项，不做扩展：

1. **登录**：账号密码登录（RSA 公钥加密密码，POST `/nuke.php` 登录后经 `login_set_cookie_quick` 捕获会话 Cookie），粘贴 Cookie 作为兜底；登录态持久化到 SharedPreferences，请求时携带。
2. **板块选择**：顶栏下拉菜单切换不同 NGA 板块（晴风村、网事杂谈、二次元国家地理、炉石传说、游戏王大师决斗、洛克王国世界）。
3. **帖子列表**：拉取指定版块帖子，展示标题/回复数/作者/最后回复时间，点击进入详情。支持下拉到底部自动加载下一页。
4. **帖子详情**：展示楼层、作者、时间、正文（支持图片显示、引用块美化）。

## 4. 页面结构

```
MainActivity (单 Activity)
├── ListScreen     帖子列表  ——  顶栏含板块选择下拉、读取剪贴板链接、设置入口
├── DetailScreen   帖子详情  ——  接收 tid 参数
└── SettingsScreen 设置页    ——  检查更新、登录 NGA 账号 / 退出登录
```

导航：ListScreen →(点击条目, 传 tid)→ DetailScreen；ListScreen →(设置入口)→ SettingsScreen。

## 5. 数据来源

NGA 论坛（GBK 编码 HTML）：

- 列表：`https://bbs.nga.cn/thread.php?fid={fid}&page={page}`
  - fid 参数：板块 ID，根据用户选择的板块动态切换
  - page 参数：页码数，从 1 开始（1、2、3、4……）
  - 首次加载时 page=1，滚动到底部时加载 page=2、3、4……
  - 解析 `<tr class="topicrow">`
  - tid：`<a href="read.php?tid=X">`
  - 标题：`<a class="topic">`
  - 回复数：`<a class="replies">`
  - 最后回复：`<a class="replydate">` / `<span class="replyer">`
- 详情：`https://bbs.nga.cn/read.php?tid={tid}`
  - 解析 `<tr class="postrow">`
  - 作者：`#postauthorN`
  - 时间：`#postdateN`
  - 正文：`#postcontentN`（解析 UBB 标签）
  - 图片标签：`[img]./xxxx.jpg[/img]` 或 `[img]xxxx.jpg[/img]`
    - 拼接规则：`https://img.nga.178.com/attachments/` + 去除 `./` 前缀的路径
    - 示例：`[img]./mon_202401/12/-7lddQ5-abc123.jpg[/img]` → `https://img.nga.178.com/attachments/mon_202401/12/-7lddQ5-abc123.jpg`
  - 引用标签：`[quote]xxxx[/quote]`
    - 渲染为：顶部显示"引用"文字标题，下方为灰色背景区域，内部展示引用的文本内容

## 7. 登录方案

参考网页端登录页 `账号操作.html` / `账号操作_files/account_copy.html` 的实现，支持账号密码登录，并保留粘贴 Cookie 作为兜底。

### 7.1 账号密码登录（主流程）

复刻网页端 `_loginUI` → `_submit` 的两步流程：

1. **加密密码**：使用 NGA 登录页内置的 RSA 公钥（2048 位），以 `RSA/ECB/PKCS1Padding`（PKCS#1 v1.5）加密明文密码，输出 Base64 字符串。对应网页端 JSEncrypt（`js_jsencrypt.js`）的 `_encrypt`，Java 与 JSEncrypt 默认填充一致。实现于 `data/RsaCipher.kt`。
2. **提交登录**：POST `https://bbs.nga.cn/nuke.php`，表单字段：
   - `__lib=login`、`__output=1`、`__act=login`、`__inchst=UTF-8`
   - `name`（账号）、`type`（账号类型：空=用户名/昵称/手机号、`mail`=邮箱、`id`=用户ID、`phone`=手机号）、`password`（RSA 加密后）
   - 响应为 `window.script_muti_get_var_store={...}` 形式的 JSON，成功时 `data[3]` 含 `uid`、`token`、`username`；失败时 `error` 数组携带错误信息。
3. **设置会话 Cookie**：用上一步拿到的 `uid` + `token`，POST `https://bbs.nga.cn/nuke.php?__lib=login&__act=login_set_cookie_quick&__output=9`（表单 `uid` + `cid=token`，对应网页端 `loginSuccess` 后父页向各域名提交表单设 Cookie）。借助 OkHttp 的 `CookieJar` 捕获服务端下发的 `Set-Cookie`，拼成 `name=value; ...` 的 Cookie 头字符串持久化。若该接口未直接返回 `Set-Cookie`，则退回用 `ngaPassportUid`/`ngaPassportCid` 构造最小 Cookie。

实现于 `data/NgaApi.kt` 的 `login(name, type, password)`，返回 `LoginResult(cookie, username)`。

### 7.2 粘贴 Cookie（兜底）

- 用户在浏览器登录 NGA 后，复制请求 Cookie（含 `ngaPassportUid`、`ngaPassportCid` 等）。
- 登录对话框内提供可折叠的「粘贴 Cookie」入口，直接保存到 `SharedPreferences`。

### 7.3 通用约定

- 所有 NGA 请求附带持久化的 `Cookie` 头与桌面 `User-Agent`。
- Cookie 存储于 `data/CookieStore.kt`：`save`/`get`/`clear`/`isLogin`，并记录账号名（仅展示用，非凭证）。
- 设置页（`ui/SettingsScreen.kt`）提供「登录 NGA 账号」入口（打开 `LoginDialog`）与「退出登录」；登录态实时展示。
- 不实现验证码、短信验证、Token 刷新、OAuth（QQ/微信/微博等）登录；如遇服务端要求验证码，按错误信息提示用户。

## 8. 分页加载

**帖子列表分页**：

- 实现方式：URL 参数 `&page=n`，n 为页码（1、2、3、4……）
- 触发时机：用户滚动到列表底部时自动加载下一页
- 加载行为：
  - 首次进入列表加载 page=1
  - 滚动到底部触发加载 page=2、3、4……
  - 新数据追加到现有列表末尾
  - 加载过程中显示底部加载指示器
- 状态管理：
  - 当前页码
  - 是否正在加载
  - 是否还有更多数据

## 9. 图片解析与显示

**UBB 图片标签解析**：

- 标签格式：`[img]./xxxx.jpg[/img]` 或 `[img]xxxx.jpg[/img]`
- 解析逻辑：
  1. 提取 `[img]` 和 `[/img]` 之间的路径
  2. 去除路径开头的 `./` 前缀（如果存在）
  3. 拼接完整 URL：`https://img.nga.178.com/attachments/` + 处理后的路径
- 示例转换：
  - `[img]./mon_202401/12/-7lddQ5-abc123.jpg[/img]` 
  - → `https://img.nga.178.com/attachments/mon_202401/12/-7lddQ5-abc123.jpg`
- 显示方式：
  - 使用 Coil Compose 库异步加载图片
  - 在帖子详情的正文中嵌入显示
  - 支持点击查看大图（可选）

## 10. 引用块美化

**UBB 引用标签解析**：

- 标签格式：`[quote]引用内容[/quote]`
- 渲染效果：
  - 顶部显示"引用"文字（小字/次要色调）
  - 下方展示一块灰色背景区域，内部包含引用的文本内容
  - 引用区域与正文其余内容之间保留适当间距
- 实现方式：
  - 在 `UbbParser` 中识别 `[quote]...[/quote]` 标签，将其解析为独立的引用节点（区别于普通文本和图片节点）
  - 在 `DetailScreen` 中针对引用节点渲染专属的 Compose 组件：
    - 上方 `Text("引用")` 作为标题标签
    - 下方 `Box` 设置灰色背景（`MaterialTheme.colorScheme.surfaceVariant` 或固定浅灰色），内部 `Text` 展示引用内容
- 嵌套引用：初版不处理嵌套 `[quote]`，内层 `[quote]` 标签作为普通文本展示
